# ADR-0002: Web Native First, No Excel Plugin

状态：Accepted

日期：2026-05-06

## 背景

SAP BPC 的 EPM Add-in / Excel 插件形态提供了强大的表格操作能力，但也带来部署、兼容性、用户培训、模板治理和插件依赖问题。BPC-KB-003 和 BPC-KB-008 已将 Excel 插件依赖识别为本项目需要规避的核心复杂形态。

本项目目标是自研 Web Native 企业级全面预算管理平台，因此预算模型管理、模板配置、填报、查询和导入流程必须在 Web 应用内完成。

## 决策

1. MVP 不开发 Excel / Office 插件。
2. 预算模板使用 Web 表格与显式行轴、列轴、筛选、单元格规则表达。
3. 填报、保存、提交、退回、通过、锁定都在 Web 应用内完成。
4. Excel 或 CSV 只可作为导入或导出的文件格式，不作为核心运行时入口。
5. 不允许用临时 HTML 或本地表格文件替代正式 React/Vite 应用。

## 理由

1. Web Native 形态更利于权限、状态、审计和版本治理。
2. 显式模板配置比隐藏在 Excel 工作簿中的逻辑更可测试、可审查。
3. Web 应用能降低客户端安装和兼容性成本。
4. 将文件格式限定为导入导出介质，可以吸收电子表格便利性而避免插件依赖。

## 后果

正向影响：

1. 用户无需安装 Office 插件即可填报预算。
2. 模板结构、字段映射和填报状态可以统一审计。
3. 前端可以围绕预算工作流建设更清晰的交互。

代价和风险：

1. Web 表格需要自行实现冻结、批量粘贴、校验提示等填报体验。
2. 重度 Excel 用户的迁移需要通过导入导出和模板易用性缓冲。
3. BUD-006 和 BUD-007 需要控制范围，避免一次性复刻 Excel 全功能。

## 非目标

1. 不实现 Office Add-in。
2. 不实现宏、VBA 或 Excel 内嵌脚本。
3. 不把 Excel 文件作为事实数据的主存储。

## 验证

后续阶段验收时，应确认：

1. 核心填报路径可在 Web 应用完成。
2. 模板配置不依赖本地 Excel 插件。
3. 文件导入导出不绕过权限、状态和审计。
