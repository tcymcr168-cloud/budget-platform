# ADR-0005: Single Primary Hierarchy for MVP

状态：Accepted

日期：2026-05-06

## 背景

SAP BPC 支持复杂成员层级、多种报表口径和丰富属性。MVP 如果直接支持多层级、多父节点、共享节点和层级版本，会显著增加成员维护、权限继承、查询汇总和历史解释成本。

PRODUCT-001 已确定首个业务场景为费用预算，BPC-KB-002 已建议 MVP 先支持单主层级。

## 决策

MVP 每个维度只支持一个主父子层级：

1. `dimension_member.parent_id` 表示主层级父节点。
2. 一个成员最多有一个父节点。
3. 事实数据原则上写入叶子成员。
4. 汇总按当前主层级计算。
5. 多层级、多父节点、共享节点、层级版本后置。

## 理由

1. 单主层级足以支撑费用预算的组织和科目汇总。
2. 权限和责任范围更容易解释。
3. 成员维护和循环校验更简单。
4. 查询汇总实现更可控。

## 后果

正向影响：

1. BUD-003 可以快速实现成员树 API。
2. BUD-008 可以基于主层级做基础汇总。
3. MVP 避免过早复制 BPC 的复杂层级能力。

代价：

1. 同一成员无法同时参与多个报表汇总口径。
2. 历史层级变化暂不具备时间版本能力。
3. 后续如需多层级，需要新增层级表或层级版本设计。

## 非目标

1. 本 ADR 不实现层级 API。
2. 本 ADR 不创建数据库表。
3. 本 ADR 不实现权限继承。
