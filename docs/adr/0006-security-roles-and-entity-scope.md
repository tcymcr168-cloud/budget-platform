# ADR 0006: 使用角色 + Entity 数据范围替代复杂多维权限矩阵

## 状态

Accepted

## 背景

预算平台需要企业级访问控制，但项目明确要求规避 SAP BPC 中难以理解和维护的复杂多维权限矩阵。当前 MVP 已具备元数据、模型、模板、填报、查询、实际数导入和预算实际差异分析，但认证、授权和数据范围尚未实现。

现有业务对象已经天然围绕 Entity、Owner、Reviewer 和 Operator 展开，因此安全模型应优先利用这些明确业务边界。

## 决策

采用“角色 + Entity 数据范围 + 流程责任人”的三层安全模型：

1. 角色决定用户能执行哪些操作。
2. Entity 数据范围决定用户能访问哪些组织或责任中心数据。
3. Owner、Reviewer、Operator 等流程责任字段决定用户能推进哪些具体任务或导入批次。

MVP 不实现 Account、Time、Category、Version 与 Entity 交叉组合的复杂权限矩阵。后续如需更细权限，只能在独立阶段评估，不得隐式混入当前安全基线。

## 结果

正向结果：

1. 权限判断可解释，便于管理员配置和审计。
2. 查询、汇总、差异分析和导入都可以围绕 Entity 范围统一收敛。
3. 避免在 MVP 阶段复制 SAP BPC 复杂权限矩阵。
4. 为后续持久化审计和生产认证预留清晰上下文。

代价：

1. 暂不支持按 Account、Category、Version 的精细组合授权。
2. 对跨组织共享预算场景支持有限。
3. 需要后续阶段在所有读写接口显式接入安全上下文。

## 后续
